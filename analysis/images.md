     Implementation Steps

     For each recommended image:

     1. Download the image from the source URL to the images/ directory
     2. Add the <image> tag to the appropriate slide in the .sld file, using the same format as existing images (e.g., <image path="filename.png" width="50%"/>)
     3. Adjust slide layout if needed -- some slides may need width/alignment adjustments to accommodate the image alongside text

     Notes

     - Some images (Karpathy tweet, Koopman slides, GitClear charts) require manual screenshots since they are behind auth walls or in PDF format
     - The terminal-based slide tool supports PNG and JPG images via <image> tags
     - Images work best at 20-50% width to not overwhelm the terminal text
     - Some slides use <screen> split layouts (like the opening with dogus.png) which could be replicated for new images

     Verification

     - After adding each image, run the keycode tool to preview the slide and verify the image renders correctly alongside the text
     - Check that image file sizes are reasonable (the existing images range from small QR codes to 3MB)
     - Ensure the visual doesn't clash with the terminal aesthetic (dark backgrounds work best)
